API Documentation

üîë Authentication Required: All blob endpoints require Basic Authentication using your AT Protocol handle (e.g., alice.bsky.social) as username and your app password.

Upload a Blob

POST /blobs/:key

Upload a new blob or update an existing one with the given key.

curl -X POST \ -H "Authorization: Basic $(echo -n 'handle:password' | base64)" \ -F "file=@myfile.jpg" \ -F "comment=Profile picture" \ -F "enableVersioning=true" \ http://localhost:3002/blobs/profile-pic

Retrieve a Blob

GET /blobs/:key

Download a blob by its key. Add ?version=id for specific versions.

curl -H "Authorization: Basic $(echo -n 'handle:password' | base64)" \ http://localhost:3002/blobs/profile-pic # Get specific version curl -H "Authorization: Basic $(echo -n 'handle:password' | base64)" \ "http://localhost:3002/blobs/profile-pic?version=2024-01-01T10:00:00.000Z"

List All Blobs

GET /blobs

Get a list of all your blobs with metadata and version counts.

curl -H "Authorization: Basic $(echo -n 'handle:password' | base64)" \ "http://localhost:3002/blobs?limit=50"

List Blob Versions

GET /blobs/:key/versions

Get version history for a specific blob.

curl -H "Authorization: Basic $(echo -n 'handle:password' | base64)" \ http://localhost:3002/blobs/profile-pic/versions

Delete a Blob

DELETE /blobs/:key

Delete a blob or specific version. Add ?version=id to delete just one version.

# Delete entire blob curl -X DELETE \ -H "Authorization: Basic $(echo -n 'handle:password' | base64)" \ http://localhost:3002/blobs/profile-pic # Delete specific version curl -X DELETE \ -H "Authorization: Basic $(echo -n 'handle:password' | base64)" \ "http://localhost:3002/blobs/profile-pic?version=2024-01-01T10:00:00.000Z"

API Information

GET /api

Get API documentation and endpoint information in JSON format.

curl http://localhost:3002/api

Health Check

GET /health

Check service status and version information.

curl http://localhost:3002/health

Response:

{ "status": "ok", "timestamp": "2025-08-11T23:35:23.075Z", "version": "1.0.0", "service": "bucket.social CAN service" }

Register Domain

POST /domains

Register a new domain for public blob hosting.

curl -X POST \ -H "Authorization: Basic $(echo -n 'handle:password' | base64)" \ -H "Content-Type: application/json" \ -d '{"domain": "yourdomain.com"}' \ http://localhost:3002/domains

List Domains

GET /domains

Get all domains registered to your account.

curl -H "Authorization: Basic $(echo -n 'handle:password' | base64)" \ http://localhost:3002/domains

Delete Domain

DELETE /domains/:domain

Remove a domain registration from your account.

curl -X DELETE \ -H "Authorization: Basic $(echo -n 'handle:password' | base64)" \ http://localhost:3002/domains/yourdomain.com

üöÄ Getting Started

  1. Create an AT Protocol account (e.g., on Bluesky)
  2. Generate an app password in your client settings
  3. Use Basic Auth with your handle and app password
  4. Start uploading and managing blobs via the API
  5. Your blobs are stored in your Personal Data Store (PDS)
  6. Optional: Set up custom domains for public blob hosting
